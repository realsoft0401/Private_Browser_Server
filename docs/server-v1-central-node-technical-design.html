四层架构

Layer 1
Platform Server
商业与平台规则层，负责主账号、商业归属、额度真相、平台放行规则和平台级审计。
层职责主账号体系 / 商业设备归属 / 平台额度真相
直接管理对象账号 / 商业机位 / 规则 / 平台审计
不直接持有Edge 资产正文 / Virtual Browser 运行细节
Layer 2
Node Server
中心节点治理层，负责节点发现、probe、bind/unbind、health/discovery 收口、节点审计和额度快照。
层职责节点中心身份 / 节点归属治理 / 中心任务事实
直接管理对象Edge Client 节点集合 / edge_clients / bind logs
不直接持有Edge 本机资产正文 / Virtual Browser 内部运行正文
Layer 3
Edge Client
边缘执行层，负责本机 Docker、slot、browser-env、本机健康与对 Virtual Browser 的生命周期管理。
层职责本机事实 / 本机执行 / 受控边缘 API
直接管理对象Docker / slot / browser-env / profile / proxy / fingerprint
不承担多节点治理 / 商业归属最终真相 / 平台额度最终真相
Layer 4
Virtual Browser
浏览器运行层，负责容器内 Chromium 运行、CDP/VNC 能力、容器内运行时环境和最终执行事实。
层职责浏览器实际运行 / CDP / VNC / 镜像 contract 执行结果
直接管理对象浏览器容器 / 浏览器进程 / 容器内运行时环境
不承担clientId / 商业归属 / 平台额度 / 中心治理状态
分层原则：按职责层级分层，不按代码量分层；同一份核心事实只允许存在一个真相源层；上层不直接绕过中间层访问下层真相源。

四层应用关系

Virtual Browser ↔ Edge Client

Virtual Browser 提供浏览器运行事实；Edge Client 管理并封装浏览器运行单元。

容器存在性 CDP / VNC 可连 load env start / stop / release

Edge Client ↔ Node Server

Edge Client 提供本机能力；Node Server 编排和治理本机能力。

/health device-info bind / unbind health / discovery

Node Server ↔ Platform Server

Node Server 负责节点中心治理；Platform Server 负责商业归属、额度真相和平台放行规则。

clientId 节点摘要 额度下发 放行规则

禁止直接链路

  • Platform Server 不直接调用 Edge Client
  • Platform Server 不直接调用 Virtual Browser
  • Node Server 不直接治理浏览器进程
  • Edge Client 不直接主导商业 clientId 和平台额度

四层数据关系

核心约束：同一份数据不得在四层同时作为真相源。

数据对象Virtual BrowserEdge ClientNode ServerPlatform Server
浏览器进程运行事实真相源管理并摘要不直接持有不持有
CDP/VNC 可用性真相源摘要与管理摘要引用不持有
本机设备事实不持有真相源摘要缓存不持有或极简摘要
slot 状态运行承载事实真相源只保存引用摘要不持有
browser-env 目录与登录态运行加载态真相源聚合摘要不持有
节点中心身份 clientId不生成不主导真相源接收并引用
平台额度不生成不生成快照缓存真相源
中心任务事实不保存不保存真相源可引用摘要
Node Server 只保存中心索引与治理摘要，不保存 Edge 本机资产正文，也不保存 Virtual Browser 容器内部运行正文。

四层状态关系

层状态类型典型状态说明
Virtual Browser浏览器运行状态容器存在 / running / CDP 可连 / VNC 可连只表达运行事实
Edge Client边缘本机状态/health、本机 Docker、slot、browser-env 生命周期只表达本机事实
Node Server中心治理状态healthy / unhealthy / stale / offline；verified / blocked表达节点治理判断
Platform Server商业状态归属有效 / 额度有效 / 平台放行表达商业与平台规则结论

需要明确区分的状态关系

  • Client /health 不等同于 Node health_status。
  • Node verified 不等同于 Platform 放行。
  • Virtual Browser 容器 running 不等同于平台业务可用。
  • Virtual Browser 可连接不等同于 Edge Client 健康。
  • Edge Client 健康不等同于 Node 节点可放行。

四层调用关系

正式主链

Platform Server → Node Server → Edge Client → Virtual Browser
主链语义 Platform 定义商业规则，Node 收口治理与准入，Edge 负责本机执行，Virtual Browser 负责实际浏览器运行。
执行原则 上层不直接绕过中间层调用下层真相源，不直接跨层写入运行事实。

反向事实上报链

Virtual Browser → Edge Client → Node Server → Platform Server
事实上收规则 Virtual Browser 的运行事实先由 Edge 吸收，再由 Node 汇总为中心节点摘要，最后按平台需要上送。
禁止事项 Virtual Browser 不直接越层上报 Platform；Platform 不直接抓取 Edge 或浏览器运行正文。

禁止调用链

禁止链路禁止原因
Platform Server → Edge Client绕过节点治理层，破坏中心审计与归属收口。
Platform Server → Virtual Browser商业规则层不应直接触达浏览器运行层。
Node Server → Virtual Browser 内部细节Node 治理对象是节点，不是浏览器进程。
Node Server → Edge 本机资产正文破坏本机资产真相源与安全边界。
Edge Client → Platform clientId/额度真相Edge 不主导商业身份与商业规则。

四层责任边界

层负责不负责
Virtual Browser实际浏览器运行、CDP/VNC 结果、容器运行事实账号归属、平台额度、中心审计
Edge Client本机事实、本机执行、边缘 API、Virtual Browser 生命周期多节点治理、商业归属、平台额度真相
Node Server节点身份、节点归属治理、节点发现与验证、health/discovery 收口、中心任务与审计本机资产正文、Virtual Browser 运行细节真相、平台额度最终规则
Platform Server商业主账号、商业设备归属、平台额度真相、平台审计与规则边缘本机执行细节、Edge 资产操作、Virtual Browser 运行细节

Node Server P0 在四层中的位置

Node Server P0 在四层中属于中间治理枢纽层。

属性定义
转译层将 Edge 的本机事实和 Virtual Browser 运行摘要转成中心可治理的节点事实
治理层承接 bind / unbind / probe / health / discovery 收口
缓冲层承接 Platform 商业规则与额度快照
收口层为后续 run 准入、env 生命周期和平台放行提供前置判断

Node Server P0 应用流程

自动发现流程

  1. Edge Client 发送 UDP beacon。
  2. Node Server 接收 beacon 并校验平台字段。
  3. Node Server 调用 /health。
  4. Node Server 调用 /api/v1/edge/device-info。
  5. Node Server 建立 discovered 内存视图。
  6. 命中已登记节点时回写最近发现与探测摘要。
收口： discovery 只形成发现态线索，不直接形成正式绑定节点。

bind 流程

  1. Platform 或管理员发起 bind。
  2. Node Server probe 目标 Edge。
  3. Node Server 执行查重与跨账号占用校验。
  4. Node Server 分配 clientId。
  5. Node Server 写入 edge_clients。
  6. Node Server 写入 bind 审计。
  7. Node Server 兼容 push clientId 到 Edge。
  8. Node Server 返回 bind 结果与节点摘要。
收口： bind 成功后节点进入正式中心身份管理；push 失败不回滚中心 bind。

unbind 流程

  1. Platform 或管理员发起 unbind。
  2. Node Server 校验节点存在且当前已绑定。
  3. Node Server 解除中心归属。
  4. Node Server 写入 unbind 审计。
  5. Node Server 调用 Edge 清空本地 node-registration.json。
  6. 清理失败时写入错误摘要与审计留痕。
收口： unbind 解除归属，不销毁原 clientId，不删除历史任务与审计。

heartbeat / health 收口流程

  1. Edge Client 发送 heartbeat。
  2. Node Server 更新心跳摘要。
  3. Node Server 刷新 discovered 视图。
  4. Node Server 按阈值推进 health_status。
  5. 必要时再次 probe 并修正治理状态。
收口： heartbeat 提供活性事实；最终中心可用性由 Node Server 统一计算，不直接等同于 Edge /health。

Node Server P0 状态设计

health_status

  • healthy
  • unhealthy
  • stale
  • offline

discovery_status

  • verified
  • blocked

discovery_reason

  • not_bound
  • probe_failed
  • ip_mismatch
  • identity_changed
  • stale

设计约束：主状态少而稳；细原因落 reason；identity_changed、ip_mismatch、discovered 不进入正式主状态枚举。

关键时序

发现与登记时序

Virtual Browser

  1. 不直接参与节点发现与登记。
  2. 仅作为后续浏览器运行承载层存在。

Edge Client

  1. 发送 UDP beacon。
  2. 响应 /health 探测。
  3. 响应 /api/v1/edge/device-info 探测。

Node Server

  1. 接收 beacon。
  2. 校验 discovery 协议字段。
  3. 调用 /health。
  4. 调用 /device-info。
  5. 写入 discovered 内存视图。
  6. 命中已登记节点时回写摘要。

Platform Server

  1. 本流程不直接参与。

bind 时序

Platform Server / Operator

  1. 发起 bind 请求。
  2. 接收 bind 结果。

Node Server

  1. probe 目标 Edge。
  2. 执行查重与归属互斥校验。
  3. 生成 clientId。
  4. 写入 edge_clients。
  5. 写入 bind 审计。
  6. 兼容 push clientId 到 Edge。

Edge Client

  1. 响应 /health 与 /device-info。
  2. 接收 push clientId。
  3. 写入本地登记留痕。

Virtual Browser

  1. 本流程不直接参与。

后续 run 准入时序

Platform Server

  1. 输出商业放行结果。
  2. 下发额度与上层业务动作。

Node Server

  1. 校验 healthy。
  2. 校验 verified。
  3. 校验已有 clientId。
  4. 校验额度快照。
  5. 调用 Edge run。

Edge Client

  1. 接收 run 请求。
  2. 选择目标 slot。
  3. 将 env 加载到 Virtual Browser。
  4. 汇总本机运行结果。

Virtual Browser

  1. 加载 env 运行材料。
  2. 启动浏览器。
  3. 提供 CDP / VNC 运行结果。

stop / ending / backup 收口时序

Platform Server / Operator

  1. 发起 stop / backup / ending。
  2. 接收最终收口结果。

Node Server

  1. 调用 Edge 动作接口。
  2. 接收动作摘要与最终结果。
  3. 更新中心状态与审计。

Edge Client

  1. 停止浏览器运行。
  2. 释放 Virtual Browser 运行态。
  3. 恢复 slot 为空白 waiting 容器。
  4. 必要时执行 backup 资产动作。

Virtual Browser

  1. 停止浏览器进程。
  2. 清理运行态。
  3. 不保留旧 env 的运行上下文。
收口约束：ending、stop、backup 成功后，slot 必须回到空白 waiting 容器；旧运行态不得遗留到下一次包加载。

说明：关键时序按“分层泳道”表达，各层只展示自己负责的动作与事实，不跨层代填语义。

逐层验收口径

层验收项
Virtual Browser浏览器成功启动；CDP 可连；VNC / WebVNC 可连；stop / ending 后可释放运行态；backup / ending 后可回到空白 waiting 容器
Edge Clientdiscovery 可用；/health 可用；/device-info 可用；边缘受控 API 可用；Virtual Browser 管理链路可用
Node Server节点发现可用；bind 可用；unbind 可用；节点治理可用；状态收口可用；中心审计可用
Platform Server商业归属可用；额度真相可用；平台放行规则可用

结论

层级结论定义
Virtual Browser负责真实浏览器运行
Edge Client负责本机边缘管理与浏览器运行封装
Node Server负责中心节点治理与边缘编排
Platform Server负责商业规则与平台最终真相

中心节点层结论：Node Server P0 的核心任务是建立稳定的中间治理层，承接节点身份、节点归属、节点健康、节点治理、节点审计和后续业务准入前提。
